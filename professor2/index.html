O QUE É A PROGRAMAÇÃO ORIENTADA A OBJETOS

A Programação Orientada a Objetos (POO) é um modelo de programação baseado no conceito de objetos que podem conter dados na forma de métodos e atributos. Esse paradigma surgiu como uma maneira de diminuir a complexidade do desenvolvimento de um software e aumentar a produtividade. Para isso, durante a programação, são desenvolvidos objetos com estruturas e comportamentos próprios que interagem entre si e executam as ações estabelecidas. Esses objetos são desenvolvidos a partir de quatro principais estruturas, sendo elas as classes, objetos, atributos e métodos.

  • Classe - A classe é um conjunto de objetos que possuem características em comum.
  • Objeto-Os objetos são elementos que pertencem a uma classe.
  • Atributos - Os atributos são características de um determinado objeto.
  • Método - É por meio dos métodos que se pode modificar os atributos de um objeto.

Para exemplificar, imagine que um Pokémon seja uma classe. Dentro dessa classe, temos vários Pokémons, como o Pikachu, ele seria o objeto. Esse objeto é uma instância da classe, logo seu comportamento e estado serão definidos por ela. Cada Pokémon possui características próprias, como seu nome e seu tipo, essas informações seriam seus atributos. Por último, temos os métodos, que nesse caso seriam as habilidades e ataques do Pokémon.




Existem quatro pilares que baseiam a Programação Orientada a Objetos, sendo eles a abstração, herança, polimorfismo e encapsulamento.

ABSTRAÇÃO

Um dos principais pilares do POO é a abstração, que consiste no processo de identificar e reduzir a complexidade do sistema, tornando mais fácil para o programador trabalhar com ele. A utilização da Abstração traz diversos benefícios ao programador e ao usuário, uma vez que ignora os detalhes irrelevantes ou secundários, permitindo criar modelos simplificados, focando nos aspectos importantes. Esse pilar também permite que detalhes de execução sejam ocultos, protegendo assim a integridade dos objetos da programação. Podemos ter como exemplo um objeto como o Mario, o personagem possui métodos como pular, coletar moeda, perder vida. A abstração permite que o programador foque nos aspectos relevantes da programação, relevando aspectos não primordiais, fazendo com que a estrutura da programação se torne mais eficiente e de fácil leitura, permitindo ao usuário uma experiência eficaz. Assim, o usuário não precisa se preocupar com o funcionamento de cada comando ele só precisa saber que ele funciona.

HERANÇA

O segundo pilar é herança, esse pilar permite que as classes compartilhem seus atributos e métodos entre outros membros da mesma classe. Existem dois tipos diferentes de Classes sendo elas a classe base (superclasse) e a classe derivada (subclasse). A superclasse também conhecida como classe pai é aquela que concede as características a subclasse chamada na programação de classe filho que herda essas características. Como exemplo imagine que você vá criar um jogo, você cria uma classe que se chama personagens dentro dessa classe os personagens correm, falam e respiram. Depois você cria subclasses com novos objetos que possuem atributos específicos como homem, herói, vilão entre outros. Essa classe herda e acrescenta os atributos da classe pai se tornando mais especializada Esse pilar é benéfico ao programado pois facilita a programação uma vez que reaproveitas códigos de comportamentos evitando a repetição de códigos semelhantes, além disso por tornar o código mais simples torna mais eficiente a manutenção do código tornando o código mais eficiente e limpo.

POLIMORFISMO

O termo polimorfismo é originário do grego, tendo seu significado poli = muitas e morphos = formas. Esse conceito permite que diferentes objetos respondam ao mesmo comando, mas de forma diferente. Como exemplo, temos uma classe chamada "Animais", dentro da qual temos gato, galinha, cachorro, entre outros animais. Eles podem executar o mesmo método, como "falar" (emitir som), entretanto, cada um executa esse método de modo diferente. O polimorfismo permite a reutilização de código, tornando a programação mais legível e organizada, facilitando a manutenção do sistema. Isso ocorre porque, ao invés de criar métodos separados para cada tipo de animal, podemos definir um método genérico "falar ()", que será implementado de forma diferente por cada subclasse de "Animais".

ENCAPSULAMENTO

O encapsulamento é o pilar que se refere a ocultar detalhes internos de um objeto, mostrando apenas a interface necessária para interagir com ele. Ou seja, os atributos e métodos de um objeto serão agrupados e protegidos em uma "caixa preta", não sendo acessíveis a menos que sejam públicos. Vamos usar um carro como exemplo. Quando você dirige um carro, não precisa conhecer todos os detalhes sobre como o motor, o sistema de aceleração e os freios funcionam. Você só precisa saber como utilizar os controles básicos, como o volante, os pedais e o painel. Com isso, o encapsulamento permite que informações desnecessárias sejam ocultadas, permitindo ao usuário não se preocupar com elementos mais complexos, apenas com os elementos básicos. O encapsulamento também traz diversos benefícios para os programadores, uma vez que ele permite a reutilização de código, tornando a programação mais limpa e eficiente.

PERGUNTAS FREQUENTES

  • Quais são as principais linguagens de programação orientadas a objetos?
    As principais linguagens de programação são o:Java, C++, C#, Python, Ruby e JavaScript.

  • Quais são as etapas básicas para desenvolver o POO?
     As etapas básicas para desenvolver um sistema POO é identificar classes e objetos, definir atributos e métodos, estabelecer relação entre as classes e utilizar dos 4 pilares (encapsulamento, abstração, herança e polimorfismo) para tornar sua programação mais eficiente.

  • Por que utilizar a programação orientada a objetos?
    A programação orientada a objetos permite diminuir a complexidade do desenvolvimento de um software e aumentar a produtividade.

  • Qual a diferença entre classe e objeto?
    Classe é a definição de um objeto. Já o objeto é a instância criada a partir dessa definição(classe).
